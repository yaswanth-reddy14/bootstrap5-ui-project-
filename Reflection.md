# Project Reflection

## Project Overview

This project was assigned to help me understand Bootstrap 5 and learn how to build responsive web pages by combining Bootstrap components into an original design. Instead of copying Bootstrap example pages, I explored different components and used them as references while creating my own layout.

I selected a software company theme called **Nexora** because it allowed me to include different sections such as services, testimonials, team members, FAQs, and a contact page in a realistic way.

---

## Development Process

I started by reading the task instructions and understanding what was expected. Then I explored the Bootstrap 5 documentation and the Bootstrap example pages to understand how different components such as the navbar, cards, forms, accordion, and grid system work.

After that, I planned the structure of the website and decided to create three pages:

- Home
- About
- Contact

I created the basic HTML files first and linked Bootstrap using the CDN. Once the basic structure was ready, I built the navigation bar so that users could move between pages easily.

Next, I worked on the Home page by adding the hero section, feature cards, statistics, services, testimonials, FAQ, and footer. After completing the Home page, I developed the About page with the company story, mission, vision, timeline, team members, and skills section. Finally, I created the Contact page with a Bootstrap form, company information, social links, FAQ section, and a map placeholder.

Once all pages were complete, I created a custom stylesheet to maintain consistent colors, spacing, typography, shadows, and hover effects throughout the website. Finally, I tested the project on desktop, tablet, and mobile screen sizes before deploying it using GitHub Pages.

## Challenges Faced

One challenge was arranging different Bootstrap components into a layout that looked consistent across all pages.

Another challenge was maintaining proper spacing and alignment while making the website responsive for different screen sizes.

I also spent time customizing Bootstrap components using CSS because I wanted the website to have its own appearance instead of looking like the default Bootstrap examples.

The contact form validation and timeline section also required some additional effort because they needed custom styling along with Bootstrap components.

---

## How I Solved Them

To keep the design consistent, I reused common styles for buttons, cards, section headings, and spacing throughout the website.

For responsiveness, I relied on Bootstrap's grid system and utility classes, then used a small amount of custom CSS to adjust layouts for smaller devices.

Whenever I was unsure about a Bootstrap component, I referred back to the official documentation and tested different approaches until I achieved the expected result.
